# 规则参考映射表（轻量扫描器 vs 参考项目）

## 说明
本项目不是直接复制 `daily_stock_analysis-main`，而是将其中适合轻量、显式、可测试扫描器的部分拆解后重构。

本映射表用于说明：
- 当前规则/计划中的规则，大致参考了完整项目的哪些分析思路
- 哪些部分已经落地
- 哪些部分仍在计划中

---

## 1. 已落地规则映射

### 1.1 趋势过滤：`close >= SMA20`
- 轻量项目用途：基础趋势确认
- 参考来源：完整项目中均线位置、趋势状态、价格位置判断思路
- 当前状态：已实现

### 1.2 趋势过滤：`SMA20 >= SMA60`
- 轻量项目用途：中期趋势确认
- 参考来源：完整项目中 MA 排列、多空结构判断
- 当前状态：已实现

### 1.3 SMA20 / SMA60 金叉确认
- 轻量项目用途：把“趋势在上方”细化为“刚发生触发”
- 参考来源：完整项目中的均线金叉/趋势启动逻辑
- 当前状态：已实现

### 1.4 Breakout 价格突破确认
- 轻量项目用途：识别突破型机会
- 参考来源：完整项目中的突破、箱体上沿、压力位突破思路
- 当前状态：已实现

### 1.5 Breakout 量能确认
- 轻量项目用途：过滤无量假突破
- 参考来源：完整项目中的量价配合、突破质量判断
- 当前状态：已实现

### 1.6 不追高约束（No-Chase Guard）
- 轻量项目用途：当价格偏离 SMA20 过大时降级
- 参考来源：完整项目中的 bias / 乖离率 / 不追高思路
- 当前状态：已实现

### 1.7 Supported Pullback（受支撑回踩）
- 轻量项目用途：识别“趋势未坏、回踩支撑、可观察再起”的结构
- 参考来源：完整项目中的回踩、支撑、缩量回踩类技术分析思路
- 当前状态：已实现

### 1.8 Pullback 量能约束
- 轻量项目用途：识别健康回踩而非放量走弱
- 参考来源：完整项目中的量能收缩、弱转强/健康整理语义
- 当前状态：已实现

### 1.9 signal_reasons / risk_flags
- 轻量项目用途：把“正向信号”和“降级风险”拆开输出
- 参考来源：完整项目中的技术结论、风险提示、解释层语义组织方式
- 当前状态：已实现

### 1.10 触发时效（Trigger Freshness）
- 轻量项目用途：避免把太旧的 breakout 继续视为新信号；当前金叉本身也只在“当日真正站上且价格日内同步确认”时记为 fresh trigger，`touching_sma60` 或“上穿但收盘没强于前收”都会显式保留为 watch-only 状态
- 参考来源：完整项目中对“启动是否新鲜、结构是否仍有效”的判断思路
- 当前状态：已实现

### 1.11 假突破 / 突破失败降级
- 轻量项目用途：识别刚突破后又跌回关键位的弱信号，并显式降级；即使同日又出现 `SMA20` 支撑回踩恢复，也仍然先保留在 `watch`
- 参考来源：完整项目中的突破质量、压力位失守、失败信号判断
- 当前状态：已实现

## 3. 明确暂不迁移的部分
以下内容虽然存在于完整项目，但当前轻量扫描器**不直接迁移**：

### 3.1 LLM 最终拍板决策
- 原因：本项目坚持“规则引擎分类，用户最终拍板”

### 3.2 通知系统 / Bot / Web UI
- 原因：当前项目定位不是平台，而是规则扫描内核

### 3.3 新闻、情绪、舆情、基本面综合智能分析
- 原因：这些更适合后续增强层，而不是当前轻量规则引擎主体

### 3.4 自动交易执行
- 原因：明确不在当前项目范围内

---

## 4. 当前结论
当前轻量项目已经借到了完整项目最重要的三类价值：
1. 趋势/触发/风险的技术分析思路
2. 量价确认与不追高语义
3. 信号与风险分离的解释结构

后续实施应继续遵循原则：
- 只迁移适合显式规则化、可测试的部分
- 不迁移大平台层、自动交易层、LLM拍板层
